📊 API ENDPOINTS SUMMARY
═══════════════════════════════════════════════════════════════════════════════

1️⃣  CREATE REFUND REQUEST
    POST /api/booking/refund-request
    Body: { booking_id, user_id, user_name, user_phone, request_type, ... }
    Response: 201 Created

2️⃣  GET ALL REFUND REQUESTS
    GET /api/booking/refund-requests?status=pending&type=refund
    Response: Array of refund requests

3️⃣  GET SPECIFIC REFUND REQUEST
    GET /api/booking/refund-request/:id
    Response: Single refund request details

4️⃣  PROCESS REFUND (Generate Transaction ID)
    POST /api/booking/refund-request/:id/process
    Body: { admin_notes: "Approved and processed" }
    Response: { status: "processed", transaction_id: "TXN_..." }

5️⃣  UPDATE REFUND STATUS
    PUT /api/booking/refund-request/:id/status
    Body: { refund_status: "rejected", admin_notes: "Reason..." }
    Response: Updated refund request
